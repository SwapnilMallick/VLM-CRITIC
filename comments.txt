Generating random actions:

1. If the actions are generated randomly, then gripper will open or close randomly for each candidate action.
This behavior is not desired. Hence, the gripper state needs to be handled separately.
Also, how to assign the correct value of the gripper state ( +1 / -1) for each random action.

2. Instead of applying uniform random sampling for generating the candidate actions, Gaussian noise is applied 
only to the translational dimensions (dx,dy,dz) while the rotational dimensions and the gripper state remain unchanged. 
Another thing to note is that the gripper state is kept open (i.e. -1) in all candidate actions for the time being.

[N(0,σ), N(0,σ), N(0,σ), 0, 0, 0, -1]

3. You can get accurate segmentation masks by adding camera_segmentations="instance" to suite.make() in robosuite. 
By adding it, the observation dictionary will add a segmentation array along with the image.

4. If the robot takes N meta (or uber??) actions from a state (may be perturbed), can a VLM identify the worst K actions based 
on some threshold.
Eg, the robot takes 4 actions (moveNorth, moveSouth, moveEast, moveWest) from a state. Each action has a value function. 
Let the four values are 0.5, 0.2, 0.0, -0.2 and the theshold be 0.1. Then the VLM should be able to filter out the actions that have an 
associated value <=0.1 (i.e. the actions with values 0.0 and -0.2).

Task: Out of n actions, find a way to flag the bad actions

1. Starting from a perturbed state take n actions. For these (n + 1) states, we have an image corresponding to each state. 
We pass these images to a segmentation model to get the features of the cube and the gripper. 
We store these features in a file (maybe JSON/YAML).

2. Given these features and the task description, we ask an LLM to generate an action rejection function that filters out the k-worst actions.
