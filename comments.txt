Generating random actions:

1. If the actions are generated randomly, then gripper will open or close randomly for each candidate action.
This behavior is not desired. Hence, the gripper state needs to be handled separately.
Also, how to assign the correct value of the gripper state ( +1 / -1) for each random action.

2. Instead of applying uniform random sampling for generating the candidate actions, Gaussian noise is applied 
only to the translational dimensions (dx,dy,dz) while the rotational dimensions and the gripper state remain unchanged. 
Another thing to note is that the gripper state is kept open (i.e. -1) in all candidate actions for the time being.

[N(0,σ), N(0,σ), N(0,σ), 0, 0, 0, -1]

3. You can get accurate segmentation masks by adding camera_segmentations="instance" to suite.make() in robosuite. 
By adding it, the observation dictionary will add a segmentation array along with the image.
